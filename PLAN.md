# Steal the Brainrot — Web 3D (PvE)

Браузерная 3D-игра про сбор брейнротов. Старт — одиночный PvE-режим:
брейнроты спавнятся волнами → игрок бежит, хватает → несёт на базу → +деньги.
PvP и мультиплеер — позже, отдельным слоем.

---

## 1. Стек

- **Three.js** — 3D-рендер (сцена, камера, свет, модели)
- **Vite** — дев-сервер + сборка (быстрый старт, HMR)
- **Vanilla JS/TS** — без фреймворка, UI поверх canvas на HTML/CSS
- **(опц.) cannon-es / rapier** — физика. На старте НЕ нужна: хватит ручных
  коллизий по расстоянию (sphere overlap).

> Решение: начинаем без физ-движка и без мультиплеера. Это даёт играбельный
> луп максимально быстро.

---

## 2. Игровой луп (core loop)

1. **Spawn** — спавнер по таймеру создаёт брейнрота в случайной точке арены.
2. **Seek** — игрок (WASD) бежит к брейнроту.
3. **Grab** — при сближении (overlap) брейнрот «прилипает» к игроку.
4. **Carry** — игрок несёт его (визуально над/за игроком), скорость чуть ниже.
5. **Deliver** — заходит в зону базы → брейнрот засчитан → +деньги, +счёт.
6. **Progress** — деньги тратятся на апгрейды (скорость, вместимость, доход).

Опционально позже: брейнроты «убегают» (AI-wander), редкость/ценность,
таймер волны, враги-воришки (PvE-ИИ перед PvP).

---

## 3. Архитектура (модули)

```
src/
  main.js            # точка входа: bootstrap, game loop (requestAnimationFrame)
  core/
    Game.js          # оркестратор: держит системы, update(dt), state
    Clock.js         # dt, фиксированный апдейт (опц.)
    Input.js         # клавиатура/мышь -> вектор намерения движения
  scene/
    SceneManager.js  # three.Scene, camera, renderer, resize, свет
    Arena.js         # пол, границы, зона базы (mesh + bounds)
  entities/
    Player.js        # mesh, движение, инвентарь (несёт N брейнротов)
    Brainrot.js      # mesh, состояние: free | carried | delivered, ценность
    Spawner.js       # таймер, пул объектов, точки спавна
  systems/
    PickupSystem.js  # проверка overlap player<->brainrot, grab/carry
    DeliverySystem.js# overlap player<->base -> начислить, очистить инвентарь
    EconomySystem.js # деньги, апгрейды
  ui/
    HUD.js           # счёт, деньги, вместимость (HTML overlay)
  assets/
    models, textures # позже; на старте — примитивы (box/sphere) + цвета
```

**Принцип:** `Game.update(dt)` по очереди дёргает системы. Сущности — просто
данные+mesh, логика в системах (лёгкий ECS-стиль, без библиотеки).

---

## 4. Технические решения

- **Камера:** third-person, следует за игроком (плавный lerp).
- **Движение:** kinematic — двигаем позицию напрямую, без физики; коллизия со
  стенами = clamp по границам арены.
- **Коллизии подбора:** дистанция между центрами < радиус → grab. Дёшево.
- **Пулинг:** Spawner переиспользует объекты (не создаём/не удаляем mesh каждый
  раз) — меньше GC-лагов.
- **Производительность:** низкополи-модели или примитивы; InstancedMesh если
  брейнротов много.

---

## 5. Этапы (вехи)

- **M0 — Каркас:** Vite + Three.js, сцена, пол, камера, рендер-цикл.
- **M1 — Движение:** игрок-куб, WASD, камера следует, границы арены.
- **M2 — Спавн+подбор:** Spawner, брейнроты-примитивы, grab по overlap.
- **M3 — База+экономика:** зона базы, доставка, деньги, HUD.
- **M4 — Контент:** модели брейнротов, редкость/ценность, апгрейды.
- **M5 — Полировка:** звук, частицы, меню, сохранение (localStorage).
- **M6+ (позже):** AI-враги → мультиплеер (Colyseus) → PvP-кража.

---

## 6. Что НЕ делаем на старте (антискоуп)

- Мультиплеер / нетворкинг
- Физический движок
- Сложный AI
- Импорт тяжёлых ассетов

Цель M0–M3: **играбельный одиночный луп за ~1–2 недели**.
